AZSlider Help

Create Slider

  • Go to Sliders page.
  • Press Create Slider .
  • We required 4 parameters: Name, Alias, Max Width, Height
    • Name is the name of slider.
    • Alias is unique string will be used as HTML id.
    • Max Width is maximum width of slider.
    • Height is height of slider.
    • Slides Layout: There are two types of layout:
      • Card: Slides will be arranged like a card pack.
      • Film: Slides will be arranged in successive row. This layout suport touch swipe navigation.

Edit Slider

  • Go to Sliders page.
  • Press button Slides of Slider you want to edit to go to Slides page.
  • On the Slides page, we have:
    • Top menu: Slider, Slide, Objects, Position, Play Slide, Play All.
    • Slides tab (to navigate between slides). Now you have 1 slide only so there is 1 tab.
    • Blank Slider in edit mode (playground).
    • Slider editor has 6 controller tabs:
      • Objects: Display the order of objects in slide, you can change the order by drag and drop.
      • Design: Manage HTML attributes of object.
      • Slide Actions: Define actions for slide.
      • Target Actions: Define actions for objects.
      • Custome Animations: where you create custom animations for slider. It can be used for other slide too.
      • CSS: Custom css for slider.

Slider Editor

  • The Concept: One slider has many slides, one slide contains many objects (Text, Image, Video...).
  • There are two slider layout:
    • Card: Slides will be arranged like a card pack.
    • Film: Slides will be arranged in successive row. This layout suport touch swipe navigation.
  • Slider editor has 6 controller tabs:
    • Objects: Display the order of objects in slide, you can change the order by drag and drop.
    • Design: Manage HTML attributes of object.
    • Slide Actions: Define actions for slide.
    • Target Actions: Define actions for objects.
    • Custome Animations: where you create custom animations for slider. It can be used for other slide too.
    • CSS: Custom css for slider.

Controller Tabs

  1. Tab Slide Actions

    • The Concept: One slide can has many actions, one action contains many commands.
    • There are 7 built-in actions:
      • in: action will be performed when slide move in the view with "next" or "back" direction.
      • out: action will be performed when slide move out the view with "next" or "back" direction.

      • next-in: action will be performed when slide move in the view with "next" direction.
      • next-out: action will be performed when slide move out the view with "next" direction.

      • back-in: action will be performed when slide move in the view with "back" direction.
      • back-out: action will be performed when slide move out the view with "back" direction.

      • swipe-complete: action will be performed when slide complete the touch swiping (film layout slider).
    • Mostly used are "in" and "out" actions. Those others will be used to create different type of slides, different effects.
    • Morover you can create custom action with any name you want.
    • There are 10 commands:
      • Bring Slide To Front: Mostly use in card layout, to bring the current slide to the top of card pack.
      • Send Slide To Back: Mostly use in card layout, to send the previous slide to the bottom of card pack.
      • Prepare Objects: Use to prepare animation of objects when they ready to show on the screen.
      • Play an Animation: Make slide animate.
      • Play an Action of Objects: Execute action of objects.
      • Play an Action of Objects After Animating: Execute action of objects when the current slide finish its animation.
      • Play Next Slide: Show next slide.
      • Play Next Slide After Animating: Show next slide when the current slide finish its animation.
      • Enable Parallax: Enable parallax effect for the current slide
      • Disable Parallax: Disable parallax effect for the current slide
    • Those commands with suffix "After Animating" must be placed right after the command "Play an Animation".

  2. Tab Target Actions

    • The Concept: Target is CSS selector string. Each target string can point to single or multiple objects of slide.
    • For example if your object has class name "test". So the target string must be ".test".
    • If more than one object has class name "test", the target string ".test" will point to multiple objects.
    • Each target can contains multiple commands. There are 18 commands:
      • Play an Animation: Make object animate.
      • Play an Action After Animating: Execute action of objects or slide when the current object finish its animation.
      • Play Slide on Click: Set event play slide for target.
      • Add Class Name: Add class name for target.
      • Add Class Name After Animating: Add class name for target when the current object finish its animation.
      • Remove Class Name: Remove class name for target.
      • Remove Class Name After Animating: Remove class name for target when the current object finish its animation.
      • Play HTML5 Video: Play video.
      • Play HTML5 Video After Animating: Play video when the current object finish its animation.
      • Pause HTML5 Video: Pause video.
      • Pause HTML5 Video After Animating: Pause video when the current object finish its animation.
      • Load Iframe: Add and load iframe url.
      • Load Iframe After Animating: Add and load iframe url when the current object finish its animation.
      • Unload Iframe: Remove iframe.
      • Unload Iframe After Animating: Remove iframe when the current object finish its animation.
      • Set HTML: Set html content for a target.
      • Set HTML After Animating: Set html content for a target when the current object finish its animation.
      • Stop Slide After Animating: Stop all animation of objects in slide.
    • Those commands with suffix "After Animating" must be placed right after the command "Play an Animation".

  3. Tab Custom Animations

    • The Concept: Animation is combine of 9 types of css transitions: move, fade, scale, skew, rotate, rotateX, rotateY, width, height and will be more in next update.
    • For example we will create an animation which move object from outside and from the top of slide and fade in.
    • Click on Custom Animations tab. Click Add Custom Animation.
    • Named this animation "move-in-from-top".
    • Choose Move in Select Type selectbox. Press Add Type.
    • We will see several inputs for Move type.
    • Object will move from x, y to x, y. We have some built-in values:
      • height, -height, width, -width, outside-top, outside-left, outside-right, outside-bottom.
    • With From X, From Y, To X and To Y value '0' mean the current position of object in slide.
    • So to make object move from outside and from the top we need:
      • From Y: outside-top.
      • To Y: 0.
    • Choose Fade in Select Type selectbox. Press Add Type. We need:
      • From: 0.
      • To: 1.
    • Object will fade from 0 to 1.
    • Now assign it to an object with command "Play an Animation" and see the result.

  4. Tab CSS

    • If you want add more css to your slider, you can go here.
    • Support Less http://lesscss.org/.
    • Support vendor-prefixed CSS3 properties with http://lessprefixer.com/.

Slider Settings

  • Slider has some settings, you can go to menu Slider > Settings.
  • Some settings:
    • Slider Max Width: Max width of slider. Slider is responsive so it can auto resize if width of parent is smaller.
    • Viewport Wrapper Max Length: Max length of viewport wrapper of slider (horizontal or vertical), it can bigger than Slider Max Width if you want to make partial slider.
    • Slides Layout: There are two layouts: card and film. If you choose film it show more 6 options:
      • Direction: Direction of film layout. Support Horizontal and Vertical direction.
      • Animation: Animation of slide when you swipe slider. Film Layout Animation only support slider with at least 3 slides.
      • Duration: Duration of animation when you released touch or mouse.
      • Easing: Easing apply for Film Layout Animation.
      • Distance Ratio: The ratio of distance between slides. Default value is 1. It will help create different effects for Film Layout Animation
      • Play Slide When: There are two options:
        • Touch Released: Play slide when you released touch or mouse.
        • Animation Completed: Play slide when animation completed.
    • Slides Transform Origin X, Slides Transform Origin Y: Transform origin x and y of slide, try different values to achives different effects.
    • Slides Perspective: 3D perspective of slides, using in 3d animation.
    • Web Font URL: Where you place your web font url which you grab from web font services like Google Font.

Create your first slide

  • We will create a simple card slide with single Hello Word! text object. The slide will starting by move from left to right. After that the text object will move from the top to center of slide.
  • Go to menu Objects > Add Text.
  • Now you have 1 text object in the playground. You can drag to positioned it.
  • Click on the text object, Click to Design tab.
  • You will see some HTML attributes of text object.
  • Type "Hello Word!" into Text textbox.
  • Click on menu Position > Horizontal and Vertical to center the text object .
  • Ok, now we have 1 slide and 1 text object in it but both of them cannot animate yet.
  • We will make slide animate first (move from left to right). Click on Slide Actions tab.
  • You will see bundle of actions and commands because when create Slide, we create some default actions. It is not important, click Delete to delete it all. We will start over to help you understand the concept of AZSlider.
  • Ok, after delete it all, click on Add Action button. You will see two inputs:
    • Action Name: Name of action of slide.
    • Command: There are 8 commands for slide action.
  • Except 6 built-in action names, you can use your custom action name to achive some effects.
  • in and out action is default action for any direction of slider (next or back) if there are no back-in, back-out, next-in and next-out settings.
  • Type "in" into Action Name textbox, this mean this action will be performed when slide move in the view in any direction (next or back).

  • The first command for action "in" is "Bring Slide To Front". Choose it in selectbox and press Add Command.
  • This command has no parameter. It mean bring slide to top of all other slides.

  • Add the second command "Prepare Objects". This command has 2 parameters:
    • Target: it can be "objects" or css selector that point to objects.
    • Action of Target: Name of action of target (ex: in, out). If we leave it blank, it will refer to current action (Here is "in").
  • Because this is "in" action so this command will be used to prepare animation settings for objects.
  • Type "objects" into Target textbox, leave Action of Target textbox blank.

  • Add the third command "Play an Animation". This command has 1 parameter:
    • Animation Name: There are some built-in animations, you can also create custom animation later.
  • Choose "short-in-left" for Animation Name selectbox. This will make slide move from left to right.
  • It is done for animating the slide. Click menu Play Slide to see your result.
  • The text "Hello word" still not moving. Now we make text object animate.

  • First, we need to named the text object (with id or class).
  • Click text object, click Design, type "t1" into Class textbox.
  • Click Target Actions. Press Add Target. Type ".t1" into Target textbox.
  • This will point action into object with class t1.
  • Press Add Action. You will see two inputs just like slide action.
  • Type "in" into Action Name textbox. Select command "Play an Animation" and press Add Command.
  • This command has 1 parameter. Choose "short-in-top".

  • Now click menu Play Slide to see your result.
  • The text is gone. What happens? The text object is prepared for animation "short-in-top". So it is moved outside view or faded to zero. It is depends on type of animation.
  • One more step to make text animate, we need set another command in slide action.
  • Click on Slide Actions, add command Play an Action of Object(s) After Animating.
  • This command has 1 parameter "Action of Target". Leave it blank to refer to the current action (Here is "in"). This command will make slide play an action of object after it animated (from left to right).
  • Now click menu Play Slide to see your result.

  • That is the concept of AZSlider, it is simple. Try to arrange actions and commands to achive different effects.

Create Grid Background

  • Grid background is combined by the rectangle blocks.
  • To create it go to menu Objects > Add Background.
  • Select an image from your computer or url.
  • Click Add Background.
  • Click on the background to select it, click on Design tab.
  • Select Grid in Background Mode selectbox.
  • You will see three new inputs:
    • Rows: number of row for the grid.
    • Cols: number of collumn for the grid.
    • Grid Scripts: a function to return data for each block of grid, so we can animate each block in different ways
      • Grid Scripts is a js function with 6 parameters: function(ri, ci, rows, cols, bw, bh){...}.
        • ri: Row index.
        • ci: Column index.
        • rows: Quantity of rows.
        • cols: Quantity of columns.
        • bw: Block width.
        • bh: Block height.
      • Grid Scripts return an array with these keys:
        • class: Class name of grid block.
        • ctn-h: Container horizontal direction (left or right).
        • ctn-v: Container vertical direction (top or bottom).
        • img-h: Image horizontal direction (left or right).
        • img-v: Image vertical direction (top or bottom).
        • trans-ox: Transforms origin x (Unit %).
        • trans-oy: Transforms origin y (Unit %).
        • ppt: 3D Perspective (Unit em).
      • Try different values of ctn-h, ctn-v, img-h, img-v, trans-ox and trans-oy to create different effects.
      • You do not need to return value for all those keys.
  • Example values for grid:
    • Rows: 4.
    • Cols: 6.
    • Grid Scripts: function(ri, ci){if(ci%2==0) return {'class': 'g1'}; else return {'class': 'g2'};}.
  • Now go to tab Objects Actions, add command "Play an Animation" for two target ".g1" with "outside-in-left", ".g2" with "outside-in-right".
  • Click Play Slide to see your result.

Objects Container

  • All objects have the same z-index is in the same container.
  • You can animate the container instead to animate each objects.
  • Container target string is '#az-ctn-{{zindex}}'. For an example with zindex = 2, target string is '#az-ctn-2'. (See Tab Target Actions)

Slide Info Area

  • Each slide has a info area outside viewport of slider.
  • You can use "Set HTML" command to set html content to slide info with target string ".az-info".
  • You can also animate the slide info with target string ".az-info". See example Partial Staff Carousel 3.

Parallax Effect

  • AZSlider support parallax effect while moving mouse over slider.
  • Each object has Z-Index property, it is also parallax level value.
  • You can enable parallax effect for the current slide by using Slide Action Command "Enable Parallax". (See Tab Slide Actions)

Custom function for Custom Animation

  • When you create Custom Animation, you will see those two parameters: Duration and Delay.
  • Except number it also accept function for more customize:
    • Function with 6 parameters: function(index, size, ri, ci, rows, cols){...}
      • index: HTML index of object.
      • size: Quantity of objects.
      • ri: Row index of block of Grid Background.
      • ci: Column inde of block of Grid Background.
      • rows: Quantity of rows of Grid Background .
      • cols: Quantity of columns of Grid Background.
  • For example: Delay: function(index, size, x, y, mx, my){return index*200;}.
  • It will set delay i*200 for objects. Object index 0 get 0ms delay, object index 1 get 200ms delay...

Make slider display many pages with only two slides using data source

  • Data Source provide data for objects in slide.
  • To save memory you can create a slider has many pages with only two slides.
  • It is the best solution when you want to display hundred of pictures.
  • We required at least two slides for this kind of slider, you can make more slides if you want.
  • Click Slider menu > Data Source. Choose JSON.
  • Data Source format: 
    [
    [{"url": "image1..."}, {"text": "text1..."}],
    [{"url": "image2..."}, {"text": "text2..."}],
    [{"url": "image3..."}, {"text": "text3..."}],
    ...
]
  • Sample data source above for slide has one image or background object and one text object.
